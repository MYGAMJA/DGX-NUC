# Hylion v6 보행학습 — 교수님 보고용 브리핑 (2026-04-17)

---

## 1. 지금 뭘 만들고 있나

**목표:** 팔 달린 인간형 로봇(Hylion v6)이 외부 충격에도 넘어지지 않고 걷게 만들기

```
Hylion v6 = BHL 다리(12 관절) + SO-ARM 팔 × 2
            └─ 이 12개 관절을 AI가 실시간으로 제어
            └─ 팔은 학습 범위 밖 (고정 자세 유지)
```

**학습 방법:** 강화학습(PPO) — 시뮬레이터 안에서 4096개 로봇이 동시에 걷기를 연습

---

## 2. 로봇이 어떻게 판단하는가

**입력 (45차원):** 목표 속도 명령 + 현재 기울기 + 관절 위치/속도 + 직전 동작

**출력 (12차원):** 다리 12개 관절의 목표 각도 (25Hz로 매초 25번 계산)

**명령 종류:**

| 명령 | 범위 |
|------|------|
| 앞뒤 속도 | -0.5 ~ +0.5 m/s |
| 좌우 속도 | -0.25 ~ +0.25 m/s |
| 제자리 회전 | -1.0 ~ +1.0 rad/s |

> "저 좌표까지 가라"는 아직 불가. 지금은 **속도 명령을 받아 걷는 엔진**만 구현된 단계.

```
  └─ 결과: 기본 보행 이미 가능한 모델 생성

  └─ Hylion v6 (팔 달린 로봇)으로 전환

                         발이 땅에 닿아도 AI가 "닿았다"는 신호를 못 받는 상황
  └─ 버그 수정 후 12000번 반복 학습

Stage C1  (4월 16일)
  └─ 팔 하중 시뮬레이션: 로봇 무게 ±0.5kg 랜덤 변동 도입
```

## 4. 지금 실패하고 있는 것 — 그리고 이유

**목표:** 외부에서 밀어도(외력) 넘어지지 않는 강건한 보행

**시도:** C1에서 한 번에 외력 ±3N 도입 → 완전 실패

| 스테이지 | 외력 | 결과 |
|---------|------|------|
| C2 | ±3N | ❌ 종료율 46.53% — 3000번 내내 계속 나빠짐 |
| C3 | ±5N | ❌ 종료율 62% |
| C4 | ±10N | ❌ 종료율 99.97% — 완전 붕괴 |

**왜 실패했나:**

```
외력을 한 번도 경험하지 않은 로봇에게
갑자기 ±3N 외력을 주입 → 뇌가 패닉 상태

학습 내내 orientation이 0.002 → 0.52로 '꾸준히' 악화
  → 회복 방향을 못 찾고 있음을 의미
  → 반복 수를 늘려도 해결 불가 (구조적 문제)
```

**핵심 원인:** 너무 큰 외력을 너무 갑자기 도입. "경험 없는 학생에게 갑자기 어려운 시험"과 동일한 원리.


## 6. 핵심 PPO 파라미터 전략

| 항목 | 값 | 이유 |
|------|-----|------|
| 학습률 (LR) | 5e-5 (고정) | 학습한 내용 잊지 않도록 |
| 업데이트 횟수 | 2~4 (단계별) | 초반엔 보수적, 후반엔 강화 |
| clip_param | 0.10~0.12 | 급격한 정책 변화 방지 |
| 성공 기준 | orientation < 15% | 이 이하면 다음 단계 진행 |
